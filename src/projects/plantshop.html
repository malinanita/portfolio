Project Description:

Elm
A full-featured e-commerce prototype for indoor plants, simulating the complete purchase flow from product browsing and filtering to cart management, checkout, and order history (excluding actual payment processing). Products, users, and orders are stored in a MySQL database, enabling secure login, personalized profiles, and the possibility for a user to view their previous orders.

I focused on secure and maintainable code by implementing password hashing, prepared statements, session handling, and input validation, while maintaining a clean architecture with strict separation of HTML and PHP and reusable templates. Orders trigger automatic email confirmations via phpMailer, enhancing the user experience.

The design is responsive, calm, nature-inspired frontend, including features such as asynchronous cart updates, category filtering, and server-rendered order detail modals. Wireframes were created in Figma to get a clear view of how features would be implemented.

This project strengthened my skills in full-stack development, database design and management, secure coding practices, and creating user-friendly, visually appealing interfaces.
